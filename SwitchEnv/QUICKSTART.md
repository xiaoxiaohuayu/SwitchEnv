# SwitchEnv 快速入门

## 🚀 快速开始

### 第一步：运行应用

```bash
# 进入项目目录
cd SwitchEnv

# 安装依赖（如果还没有安装）
pnpm install

# 启动开发模式
pnpm dev
```

应用启动后，你会看到一个美观的环境变量管理界面。

### 第二步：创建你的第一个配置

1. **点击左上角的"新建配置"按钮**
   - 输入配置名称，例如："开发环境"
   - 可选：添加描述，例如："本地开发使用的环境变量"
   - 点击"确定"

2. **配置已创建！** 你会在左侧列表中看到新创建的配置。

### 第三步：添加环境变量

#### 方式一：手动添加
1. 在左侧列表中选择刚创建的配置
2. 点击右侧的"添加变量"按钮
3. 在表格中输入：
   - **变量名**: `NODE_ENV`
   - **变量值**: `development`
4. 点击回车或点击其他地方，变量会自动保存

#### 方式二：使用快捷模板
1. 点击"使用模板"按钮
2. 选择一个模板，例如："Node.js"
3. 模板中的所有变量会自动添加到你的配置中
4. 你可以根据需要修改这些变量的值

### 第四步：启用配置

1. 在左侧列表找到你想使用的配置
2. 点击该配置上的"启用"按钮
3. 看到绿色的勾选标记？恭喜！该配置已激活
4. 环境变量已经被写入到 `.env` 文件中

### 第五步：导出配置（可选）

如果你想分享配置或备份：

1. 选择要导出的配置
2. 点击"导出"按钮
3. 选择导出格式：
   - **JSON**: 包含完整的配置信息（推荐备份使用）
   - **.env**: 标准环境变量文件（推荐与其他项目共享）
4. 选择保存位置，完成！

## 💡 实用技巧

### 快速切换环境

假设你有多个环境配置（开发、测试、生产）：

1. 在左侧列表中点击要切换的配置
2. 点击"启用"按钮
3. 一秒切换完成！

### 批量导入变量

如果你已经有一个 `.env` 文件：

1. 点击"导入"按钮
2. 选择 ".env" 格式
3. 选择你的 `.env` 文件
4. 所有变量会自动导入并创建新配置

### 复用模板

使用预设模板可以大大提高效率：

- **Node.js 项目**: 包含 NODE_ENV, PORT, DEBUG 等常用变量
- **React 项目**: 包含 REACT_APP_* 系列变量
- **Vue 项目**: 包含 VUE_APP_* 系列变量
- **数据库配置**: 包含数据库连接相关变量
- **API 配置**: 包含 API 地址、超时等配置

## 📋 常见场景示例

### 场景 1: 前端开发环境管理

```
配置名称: 本地开发
变量:
- VUE_APP_API_URL = http://localhost:3000/api
- VUE_APP_ENV = development
- VUE_APP_DEBUG = true

配置名称: 测试环境
变量:
- VUE_APP_API_URL = https://test-api.example.com/api
- VUE_APP_ENV = testing
- VUE_APP_DEBUG = false

配置名称: 生产环境
变量:
- VUE_APP_API_URL = https://api.example.com/api
- VUE_APP_ENV = production
- VUE_APP_DEBUG = false
```

**使用方法**: 需要切换到哪个环境，就启用对应的配置即可。

### 场景 2: 数据库连接切换

```
配置名称: 本地数据库
变量:
- DB_HOST = localhost
- DB_PORT = 3306
- DB_USERNAME = root
- DB_PASSWORD = 123456
- DB_DATABASE = dev_db

配置名称: 远程数据库
变量:
- DB_HOST = remote.db.example.com
- DB_PORT = 3306
- DB_USERNAME = app_user
- DB_PASSWORD = secure_password
- DB_DATABASE = prod_db
```

**使用方法**: 本地开发时启用"本地数据库"，部署时启用"远程数据库"。

### 场景 3: 多项目管理

```
配置名称: 项目A - 开发环境
配置名称: 项目A - 生产环境
配置名称: 项目B - 开发环境
配置名称: 项目B - 生产环境
```

**使用方法**: 切换项目时，同时切换对应的环境配置。

## ⚠️ 注意事项

1. **启用配置后**: 环境变量会写入到应用数据目录的 `.env` 文件中
2. **删除前确认**: 删除配置或变量是不可恢复的操作
3. **定期备份**: 重要配置建议导出为 JSON 文件备份
4. **敏感信息**: 虽然数据存储在本地，但仍需注意不要将包含密码的配置文件分享给他人
