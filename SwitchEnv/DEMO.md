# SwitchEnv 使用演示脚本

这是一个演示脚本，展示 SwitchEnv 的典型使用流程。

## 场景：为一个 Vue 项目配置开发和生产环境

### 步骤 1: 创建开发环境配置

1. 启动应用 `pnpm dev`
2. 点击"新建配置"
3. 输入：
   - 配置名称: `Vue 项目 - 开发环境`
   - 描述: `本地开发使用的环境变量配置`
4. 点击"确定"

### 步骤 2: 使用模板快速添加变量

1. 选中刚创建的"Vue 项目 - 开发环境"
2. 点击"使用模板"按钮
3. 选择"Vue"模板
4. 模板变量自动添加：
   ```
   VUE_APP_API_URL = http://localhost:3000/api
   VUE_APP_ENV = development
   VUE_APP_VERSION = 1.0.0
   VUE_APP_DEBUG = true
   ```

### 步骤 3: 修改变量值

1. 在表格中修改变量值：
   - `VUE_APP_API_URL` → `http://localhost:8080/api`
   - `VUE_APP_VERSION` → `1.0.0-dev`
2. 自动保存，无需手动操作

### 步骤 4: 添加自定义变量

1. 点击"添加变量"
2. 添加新变量：
   ```
   VUE_APP_TITLE = 我的 Vue 应用
   VUE_APP_LOCALE = zh-CN
   ```

### 步骤 5: 创建生产环境配置

1. 点击"新建配置"
2. 输入：
   - 配置名称: `Vue 项目 - 生产环境`
   - 描述: `生产部署使用的环境变量配置`
3. 点击"确定"

### 步骤 6: 手动添加生产环境变量

为生产环境添加以下变量：
```
VUE_APP_API_URL = https://api.myapp.com/api
VUE_APP_ENV = production
VUE_APP_VERSION = 1.0.0
VUE_APP_DEBUG = false
VUE_APP_TITLE = 我的 Vue 应用
VUE_APP_LOCALE = zh-CN
```

### 步骤 7: 激活开发环境

1. 在左侧列表选择"Vue 项目 - 开发环境"
2. 点击"启用"按钮
3. 看到绿色勾选标记，环境已激活
4. 环境变量已写入 `.env` 文件

### 步骤 8: 导出配置备份

1. 选择"Vue 项目 - 开发环境"
2. 点击"导出"按钮
3. 选择"JSON"格式
4. 保存为 `vue-dev-config.json`

### 步骤 9: 切换到生产环境

1. 在左侧列表选择"Vue 项目 - 生产环境"
2. 点击"启用"按钮
3. 环境瞬间切换！

## 场景：团队协作 - 导入队友的配置

### 场景描述
你的队友分享了他的开发环境配置文件 `team-dev.json`

### 操作步骤

1. 点击"导入"按钮
2. 选择"JSON"格式
3. 选择队友分享的 `team-dev.json` 文件
4. 配置自动导入到列表
5. 根据需要修改配置中的变量值

## 场景：从现有 .env 文件迁移

### 场景描述
你已经有一个项目的 `.env` 文件，想导入到 SwitchEnv

### 操作步骤

1. 点击"导入"按钮
2. 选择".env"格式
3. 选择你的 `.env` 文件
4. 所有变量自动导入
5. 为新导入的配置设置名称和描述

## 实用技巧

### 技巧 1: 快速复制配置

如果你想基于现有配置创建新配置：

1. 选择要复制的配置
2. 点击"导出"→ 选择"JSON"
3. 保存文件
4. 点击"导入"→ 选择刚保存的 JSON
5. 修改新配置的名称和变量

### 技巧 2: 批量管理多个项目

为不同项目创建配置：

```
项目A - 开发环境
项目A - 测试环境
项目A - 生产环境

项目B - 开发环境
项目B - 测试环境
项目B - 生产环境
```

需要切换项目时，同时切换对应的环境配置。

### 技巧 3: 模板组合使用

你可以多次使用模板：

1. 先选择"Vue"模板添加 Vue 相关变量
2. 再选择"Database"模板添加数据库配置
3. 再选择"Redis"模板添加缓存配置

一个配置中可以包含来自多个模板的变量。

### 技巧 4: 定期备份

建议每周导出一次重要配置：

1. 选择配置
2. 导出为 JSON
3. 保存到 Git 仓库或云盘

## 常见使用模式

### 模式 1: 本地开发 ↔ 远程测试

```
本地开发环境（默认激活）
├─ API_URL = localhost:3000
├─ DB_HOST = localhost
└─ DEBUG = true

远程测试环境
├─ API_URL = test.api.com
├─ DB_HOST = test.db.com
└─ DEBUG = false
```

需要联调时，一键切换到"远程测试环境"。

### 模式 2: 多数据库切换

```
本地 MySQL
├─ DB_TYPE = mysql
├─ DB_HOST = localhost
└─ DB_PORT = 3306

本地 PostgreSQL
├─ DB_TYPE = postgres
├─ DB_HOST = localhost
└─ DB_PORT = 5432
```

测试不同数据库时快速切换。

### 模式 3: 多语言环境

```
中文环境
├─ LOCALE = zh-CN
├─ CURRENCY = CNY
└─ TIMEZONE = Asia/Shanghai

英文环境
├─ LOCALE = en-US
├─ CURRENCY = USD
└─ TIMEZONE = America/New_York
```

国际化测试时切换语言环境。

## 注意事项

1. **激活配置前检查**: 确认变量值是否正确
2. **敏感信息保护**: 导出配置时注意不要包含密码等敏感信息
3. **定期备份**: 重要配置建议备份
4. **命名规范**: 建议使用清晰的命名，如"项目名-环境名"
